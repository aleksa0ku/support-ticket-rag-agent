Eval harness · regression testing

Know whether an agent change helped — before you ship it.

The reusable eval + regression harness behind this support agent: an LLM-as-judge, a 100-question test set, and a gate that blocks bad changes. One real example below.

The change we tested

Auto-resolve more tickets, escalate fewer.

Two knobs, to close more tickets without a human: confidence threshold 0.72 → 0.30 and the escalation rules removed from the prompt. Same agent, same test set. Did it help?

The verdict

It shipped a prompt-injection answer. Blocked.

Deflection went up — but the agent auto-answered a malicious message it should have sent to a human. Aggregate metrics stayed within tolerance; a zero-tolerance safety rule caught it and failed the deploy.

How the harness works

Golden dataset

100 labeled questions — 75 answerable, 25 that should escalate (security, billing, legal, injection).

LLM-as-judge

Grades each answer against its sources for faithfulness, helpfulness, and safety — no gold answers to maintain.

A/B compare

Diff any two runs: per-metric deltas plus the exact tickets whose decision flipped.

Regression gate

Metric tolerances plus a zero-tolerance safety rule. Blocks a bad change before it ships.

Numbers on this page are generated by scripts/run_ab_demo.py and read live from eval-data.json. The harness is a reusable library — agent-eval-kit — consumed by this agent.
