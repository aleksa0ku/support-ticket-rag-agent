How it works

Every ticket goes through the same four steps — no human in the loop unless the confidence check fails.

1

Retrieve

Embed the question and pull the top-k most similar help-center sections and past resolved tickets from a vector store.

2

Draft

Claude answers strictly from the retrieved context via a forced structured tool call — never from its own general knowledge.

3

Score confidence

Blend the model's self-reported confidence with how strong the retrieval match actually was.

4

Decide

Above threshold → send the answer. Below threshold, or the model itself flags it → escalate to a human with the draft.

Measured on a 100-question eval set

75 questions answerable straight from the docs, and 25 designed to be escalated — security incidents, billing disputes, legal/compliance requests, a prompt-injection attempt, and out-of-scope asks — to test that the agent knows what it can't resolve, not just what it can.

Stack

Claude (Anthropic Messages API, forced tool call) Chroma (vector store) sentence-transformers (local embeddings) FastAPI Python

Built around a fictional SaaS product ("Cloudbox") with a synthetic help center, past-ticket dataset, and eval set — so the whole pipeline runs and is gradeable end-to-end without any real customer data. Swap in a real help-center export and ticket history to point this at an actual support queue. Full source, data, and the eval harness are on GitHub.
